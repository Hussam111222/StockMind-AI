# StockMind AI — Backend

نظام خلفي كامل يحسب يوميًا "إشارات" لكل سهم بالقائمة (صعود / تحذير) بالاعتماد على:
- بيانات سعرية (حقيقية عبر Alpha Vantage إذا وفرت مفتاح، أو محاكاة واضحة المصدر إذا ما وفرت)
- مؤشرات فنية محسوبة محليًا (RSI, SMA50/200, Momentum)
- بحث أخبار حقيقي حي عبر أداة web search في Claude API
- تحليل AI يدمج كل هذا في قراءة نصية مختصرة

## التشغيل محليًا

```bash
cd stockmind-backend
npm install
cp .env.example .env
# افتح .env وحط ANTHROPIC_API_KEY (إلزامي) و ALPHA_VANTAGE_KEY (اختياري)

npm run refresh   # يشغّل التحديث اليومي مرة وحدة يدويًا عشان تعبّي البيانات أول مرة
npm start         # يشغّل السيرفر + الجدولة اليومية التلقائية
```

افتح `http://localhost:3000` — بتلقى الداشبورد.

## البنية


## كيف تُحسب الإشارة (Composite Score)

- **Technical score** (وزن 70%): مبني على RSI، تقاطع SMA50/200 (Golden/Death Cross)، وMomentum لآخر 20 يوم.
- **News score** (وزن 30%): من قراءة الأخبار الحية (Positive/Negative/Mixed/Quiet).
- composite ≥ 30 → **"rising"** (إشارة صعود) · composite ≤ -30 → **"caution"** (إشارة تحذير) · غير كذا → **"neutral"**.

كل هذا شفاف وقابل للتعديل من `src/scoring.js` — عدّل الأوزان والعتبات كما يناسبك.

## مزوّد الذكاء الاصطناعي (مجاني أو مدفوع)

النظام يدعم مزوّدين، تختار وحد بـ `.env`:

| | Gemini (مجاني) | Anthropic (مدفوع) |
|---|---|---|
| التكلفة | مجاني فعليًا، بدون بطاقة | ~$5-10/شهر لاستخدامك |
| مفتاح | https://aistudio.google.com/app/apikey | https://console.anthropic.com |
| بحث ويب حي للأخبار | ❌ (يعتمد فقط على Finnhub لو مُفعّل) | ✅ مدمج |
| الجودة | جيدة جدًا لهالاستخدام | أعلى شوي بالتحليل الدقيق |

**التوصية:** حط `GEMINI_API_KEY` + `FINNHUB_API_KEY` مع بعض = نظام مجاني 100% كامل الوظائف (أخبار حقيقية من Finnhub + تحليل AI من Gemين).

لو ما حطيت `AI_PROVIDER` صراحة، النظام يختار تلقائيًا: Gemini لو مفتاحه موجود، وإلا Anthropic.

## مصدر الأخبار

`src/newsProvider.js` يجيب أخبار الشركة من **Finnhub** (مصدر مالي متخصص، فيه خطة مجانية) — إذا حطيت `FINNHUB_API_KEY` بـ `.env`، النظام يستخدمها كمصدر أساسي أدق وأكثر ثبات. إذا ما حطيت المفتاح، يرجع تلقائيًا لبحث الويب الحي عبر Claude (نفس الطريقة القديمة) — يعني يشتغل بالحالتين.

سجّل مفتاح مجاني من: https://finnhub.io/register (تفعيل فوري، الخطة المجانية تكفي لمتابعة عدة أسهم يوميًا).

## طبقة التنبؤ الإحصائي (جديد)

فيه الآن نموذج تعلّم آلي حقيقي (Logistic Regression بسيط) في `src/mlModel.js`:
- يتدرب أول مرة على تاريخ السهم نفسه (RSI، Momentum، فجوة SMA، عائد اليوم → يتوقع هل الإغلاق القادم أعلى أو أقل)
- **يتعلّم من أخطائه فعليًا**: كل يوم يشوف توقعه السابق كان صح ولا غلط، ويسوي خطوة تدرّج (gradient step) على الأوزان بناءً على الخطأ — تحديث تراكمي حقيقي مو نص مكتوب
- يحتفظ بسجل أداء متجدّد (آخر 60 توقع) ويحسب دقته التاريخية الفعلية، ويعرضها بصراحة بالداشبورد

**كن صريح مع نفسك بخصوص توقعه:** توقّع اتجاه سهم ليوم واحد صعب جدًا إحصائيًا حتى لنماذج الصناديق الاحترافية — أي دقة قريبة من 50% تعادل تقريبًا رمي عملة. النموذج مصمم يكون شفاف بدقته الحقيقية (يعرضها لك رقم واضح) بدل ما يدّعي شي مب صحيح. استخدمه كمؤشر إضافي ضمن الصورة الكاملة (فني + أخبار + إحصائي)، مو كمصدر وحيد للقرار.

وزن الـ composite الحالي: 55% فني + 25% أخبار + 20% إحصائي — عدّله من `src/scoring.js`.

## معدّل التحديث

فيه الآن جدولتان منفصلتان (بالأسفل بالتفصيل)، تقدر تعدّل التوقيت من `.env`:

- **`REFRESH_CRON`** (افتراضي: `0 6 * * *` = مرة يوميًا الساعة 6 صباحًا): تحديث كامل — سعر + مؤشرات فنية + تقدّم النموذج الإحصائي (تعلّم من أمس + توقّع اليوم) + أخبار + تحليل AI.
- **`NEWS_CRON`** (افتراضي: `0 */3 * * *` = كل 3 ساعات): تحديث أخف — يعيد فحص الأخبار فقط ويعيد حساب الـ composite score وتحليل AI، **بدون** إعادة جلب الأسعار ولا لمس النموذج الإحصائي (لأن بيانات السعر اليومية أصلًا ما تتغير خلال اليوم، والنموذج الإحصائي مصمم يتقدّم مرة وحدة باليوم بمنطقه).

هذا التقسيم يخليك "محدّث" بشكل حقيقي بين تحديث وتحديث بدون ما تستهلك حصة Alpha Vantage المجانية (25 طلب/يوم) على تحديثات ما لها داعي.

تشغيل يدوي من التيرمنال:
```bash
npm run refresh        # تحديث كامل
npm run refresh:news   # تحديث أخبار فقط
```

## النشر (Deployment)


يشتغل على أي منصة تدعم Node.js طويل التشغيل (لأنه يحتاج cron يشتغل داخل العملية نفسها):
- **Render / Railway / Fly.io**: أسهل خيار — انشر كـ Web Service عادي، حط متغيرات البيئة من `.env.example`.
- **VPS عادي**: `pm2 start server.js` أو Docker.
- ⚠️ **تجنب Vercel/Netlify serverless** لهذا التصميم تحديدًا — الدوال هناك لا تبقى تشتغل بالخلفية لتشغيل cron داخلي؛ لو تبي تنشر عليهم لازم تستبدل `node-cron` بجدولة خارجية (Vercel Cron / GitHub Actions) تنادي `POST /api/refresh`.

## ⚠️ ملاحظة تنظيمية مهمة

هذا النظام يعطي "إشارات" مبنية على بيانات — وليس توصيات استثمارية مضمونة. إذا تخطط تعرضه كمنتج/خدمة لأشخاص غيرك (مو استخدام شخصي فقط)، قدّم "نصائح" عن أسهم معينة قد يعتبر قانونيًا **نشاط استشارة مالية منظّم** في أغلب الدول (مثلاً هيئة السوق المالية CMA بالسعودية، أو SEC/FINRA بأمريكا)، ويتطلب ترخيصًا. راجع مختص قانوني قبل الإطلاق التجاري. الداشبورد فيه إخلاء مسؤولية ظاهر دائمًا، لكن هذا لا يغني عن الترخيص إذا كان الاستخدام تجاري.

## حدود Alpha Vantage المجاني

النسخة المجانية محدودة بحوالي 25 طلب/يوم — كافية لقائمة صغيرة (8 أسهم) تتحدث مرة يوميًا، لكن لو زدت القائمة بكثير بتحتاج خطة مدفوعة أو مزود ثاني.
